# 엑셀 API 명세 형식 가이드

## 탭 구분 컬럼 구조

엑셀에서 복사한 텍스트는 탭(`\t`)으로 구분된다.

### 컬럼 매핑 (0-indexed)

| 인덱스 | 컬럼명 | 설명 |
|--------|--------|------|
| 0 | 순번 | 숫자, 헤더/섹션 제목행 스킵 기준 |
| 1 | 구분 | `Header` 또는 `Body` |
| 2 | 한글명 | 필드 한글명 |
| 3 | 영문명(top-level) | 최상위 필드 영문명 (snake_case) |
| 4 | 영문명(child) | 자식 필드 영문명 (한 단계 들여쓰기) |
| 5 | 영문명(grandchild) | 손자 필드 영문명 (두 단계 들여쓰기) |
| ... | 데이터타입 | String / List / Object / Integer 등 |
| ... | 길이 | 최대 길이 |
| ... | 필수여부 | Y / N |
| ... | 구분자 | 특수 구분자 (`;`, `\|` 등) |
| ... | 부모필드 | (미사용) |
| ... | 반복여부 | Y / N |
| ... | 입력값(예시) | 예시값 (줄바꿈 포함 가능) |
| ... | 비고 | 설명 |

### 계층 구조 판별 규칙

```
최상위 필드:  index[2]=한글명, index[3]=영문명
자식 필드:    index[2]=한글명, index[3]=빈값, index[4]=영문명
손자 필드:    index[2]=한글명, index[3]=빈값, index[4]=빈값, index[5]=영문명
```

자식 필드는 직전 `List` 또는 `Object` 타입 필드의 하위 항목이다.

## 섹션 구분 패턴

- `응답(Response) 인터페이스 항목` → Response 섹션 시작
- `요청(Request) 인터페이스 항목` → Request 섹션 시작

탭이 없는 행(제목행)으로 섹션을 구분한다.

## 예시값 파싱 규칙

| 예시값 텍스트 | 추출값 |
|---------------|--------|
| `5` | `"5"` |
| `"5\nnull"` | `"5"` (첫 줄 사용) |
| `"9000\n0\nnull : ..."` | `"9000"` (첫 줄 사용) |
| `"0 : 무료"` | `"0"` (`:` 앞부분) |
| `null` (단독) | `null` |
| `202603131459` | `"202603131459"` |
| `입퇴원확인서;3\|진료비;1` | `"입퇴원확인서;3\|진료비;1"` |

## Mock 생성 규칙

### 타입별 처리

| 타입 | 처리 방식 |
|------|-----------|
| `String` | 예시값 사용, 없으면 필드명 기반 기본값 |
| `Integer`, `Long`, `Number` | 예시값 사용, 없으면 숫자 문자열 |
| `List` | 자식 필드로 구성된 배열 (1개 항목) |
| `Object` | 자식 필드로 구성된 객체 |
| `Boolean` | `true` |

### 필드명 기반 기본값 (예시값 없을 때)

| 패턴 | 기본값 |
|------|--------|
| `*date*`, `*time*`, `*dt*` | `"202603231200"` |
| `*amount*`, `*price*`, `*count*` | `"1000"` |
| `*code*`, `*cd*` | `"01"` |
| `*id*`, `*no*`, `*key*` | `"12345"` |
| `*name*`, `*nm*` | `"샘플명"` |
| `*yn*`, `*flag*` | `"Y"` |
| 기타 | `"sample"` |

### 필수여부 처리

- `필수여부=N` + 예시값 없음 → `null`
- `필수여부=N` + 예시값 있음 → 예시값 사용 (실제 값 반환 케이스)

## 복수 응답 시나리오

같은 API에 Response가 여러 개인 경우 (예: 목록 조회 / 상세 조회) 각각 별도 JSON으로 생성한다.
